# The Steelbore Standard ‚Äî Changelog

Version history for the `spacecraft-standard-constitution` skill encoding of The Steelbore
Standard. The current version is recorded in the `SKILL.md` masthead; this file
holds the full history so it doesn't load into agent context on every skill
activation. The published Standard document (`standard/The_Steelbore_Standard.md`)
keeps its own inline changelog as the canonical record.

- **v1.33 (2026-07-24):** **¬ß18 added:** Accessibility (Opt-In Mode Layer) ‚Äî accessibility codified as a first-class, auditable chapter covering CLI, TUI, and GUI, superseding ¬ß13's design-system framing where the two overlap. Two-sided rule: **mandatory for developers to implement, off by default for users**, with no new-projects-only phase-in. ¬ß18.1 defines a single activation toggle with four sources and explicit precedence (`--accessible` > `SPACECRAFT_A11Y` > config > auto-detect hints such as `TERM=dumb`/`NO_COLOR`/`GTK_MODULES=‚Ä¶gail:atk`); unset everywhere means standard `Steelbore` rendering, unchanged. ¬ß18.2 splits CLI/TUI rules into always-on requirements (status never color-only ‚Äî `[OK]`/`[ERROR]`/`[WARN]`/`[INFO]`; stdout/stderr separation; `NO_COLOR` family honored) and accessible-mode requirements (no animation, no decorative art, linear output, tabular fallback, legible prompts), plus ¬ß18.2.3 TUI linear mode and a required non-interactive CLI path. ¬ß18.3 mandates **AccessKit** for Rust/custom-drawn GUI (UIA / NSAccessibility / AT-SPI) with `GtkAccessible`, Flutter `Semantics`, and `QAccessible` for toolkit-native paths, and requires explicit accessible names and roles. ¬ß18.4 requires verification by real screen reader (Orca/NVDA/VoiceOver) and `espeak-ng`, and a dated remediation entry in `PROJECTS.md` for any project not yet conforming. **¬ß11.1.1 added:** accessibility theme variants as *additive siblings* ‚Äî `steelbore` remains the sole default; `steelbore-high-contrast` lifts only `accent` (`#4B7EB0` ‚Üí `#7FAEDC`, 8.73:1) and `error` (`#FF5C5C` ‚Üí `#FF8080`, 8.41:1) so all six tokens reach ‚â•7:1 on Void Navy, with the other four ¬ß11 values verbatim; `steelbore-mono` uses 4-bit ANSI to defer to the user's terminal palette. Void Navy remains the background in every variant. **¬ß11 clarified:** the AA guarantee covers foreground-on-Void-Navy only ‚Äî 9 of 10 inter-token pairs fall below 3:1 (Molten Amber on Red Oxide 1.13:1; Radium Green on Liquid Coolant 1.01:1), so text on a palette-colored fill now requires its own pair verification, and color may never be the sole carrier of meaning. **¬ß10 extended:** bindings must be user-remappable, and NVDA/Orca/VoiceOver modifier chords must not be captured. **¬ß13 retargeted:** WCAG 2.1 AA ‚Üí **WCAG 2.2 Level AA**, with **EN 301 549 clause 11 (non-web software)** adopted as the normative anchor for CLI/TUI. **¬ß18.5 added:** Games Carve-Out ‚Äî projects registered as games are **exempt from ¬ß18 in full and from ¬ß10 in full**; accessibility in a game is optional, nothing is enforced, and its absence is never a compliance failure. ¬ß18 assumes a character grid or a widget tree with roles and names, which games (real-time simulations rendering custom non-widget interfaces) satisfy neither of; the techniques that suit them ‚Äî remappable controls, colorblind-safe signalling, subtitles, difficulty options ‚Äî are a different discipline. ¬ß18.5.1 identifies games by **declaration plus registry**, mirroring the ¬ß5.3 general-use carve-out: a `README.md` declaration alongside the ¬ß5.2 posture section, plus a registry table (today naming **Ironway**), kept in sync with `PROJECTS.md` and ¬ß2.1. ¬ß18.5.2 offers a recommended list a game may decline outright (remappable controls; leave `Insert`/`CapsLock`/`KP_Insert`/`Ctrl`+`Option` free for NVDA/Orca/VoiceOver; colorblind-safe signalling; subtitles; reduced motion). ¬ß18.5.3 asks only that a toggle a game *chooses* to ship use the ¬ß18.1 and ¬ß11.1.1 names ‚Äî constraining naming, never requiring a feature. **¬ß10 scope** narrowed accordingly: registered games are exempt in full, including the pre-existing CUA and Vim rows, since modal editing and text-editor chords are a poor fit for real-time play. **¬ß18.4:** registered games owe no remediation entry, because they owe no conformance. Compliance-checklist gains a ¬ß18 bullet; ¬ß10 and ¬ß13 bullets extended; ¬ß10 and ¬ß18 bullets marked N/A for registered games. Skill Cross-References gains a `spacecraft-accessibility-support` row.
